Synchronous Shopping Problem Solver

A continuación se presenta un formulario sencillo, el cual se debe llenar de acuerdo a las siguientes especificaciones para resolver el problema de HackerRank estipulado en este link.

  • El campo Parameters debe contener tres numeros separados por comas SIN espacios (p. ej. 5,5,5). Estos representan el número nodos, arcos y peces respectivamente.
  • El campo Shopping Centers debe contener N arreglos de números, donde N es el número de nodos ingresado en Parameters. Los arreglos deben estar separados por dash (-), y los números separados por comas (p. ej. 1,1-1,2-1,3-1,4-1,5). Para cada arreglo, el primer número representa la cantidad de tipos de peces disponibles en el nodo, y los números siguientes representan los tipos de peces.
  • El campo Roads debe contener M arreglos de números, donde M es el número de arcos ingresados en Parameters. Los arreglos deben estar separados por dash (-), y los números separados por comas (p. ej. 1,2,10-1,3,10-2,4,10-3,5,10-4,5,10). Para cada arreglo, los números representan el nodo origen, nodo destino y distancia entre los nodos respectivamente.

No seguir las condiciones anteriores dará una respuesta no valida por parte del solver.
